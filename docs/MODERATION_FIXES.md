# Доработки по замечаниям модерации (попытка 2)

## Что сделать в кабинете vendors (вручную)

| № | Замечание модератора | Действие |
|---|----------------------|----------|
| 1 | Юридическое название | **Очистить поле** — приложение не в Реестре ПО |
| 7 | Scope «Пользователи» | **Снять полный scope**, оставить только: crm, userfieldconfig, disk, bizproc, placement |
| 8 | Мобильная витрина | **Снять** «Совместимо с мобильной витриной» |

## Что исправлено в коде (деплой на up-fix.ru)

1. **REST-проверка** — устойчивый ответ API, bizproc помечен как опциональный, ошибки показываются в интерфейсе.
2. **Создание UF-полей** — fallback через `userfieldconfig.add`, ошибки смарт-процесса не блокируют поля сделок.
3. **Scope user** — удалён из приложения; подписант берётся из поля «ФИО подписанта» в настройках.
4. **Иконка** — `public/assets/ooofix_xmlupd_logo.png` 650×650 (было 135×135).
5. **Страницы support/demo** — лендинг именно про приложение «Генерация XML (УПД)» с формой связи.
